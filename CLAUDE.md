# CLAUDE.md — Инструкции для Claude Code

## Что это за проект

Консольный инструмент (CLI) для обработки анкет транспортных обследований населения.
На вход подаются: база анкет (xlsx), границы территорий (geojson), справочники (xlsx).
На выход: отфильтрованные данные, статистика, OD-матрицы, GIS-файлы, готовый QGIS-проект.

## Контекст предметной области

Транспортные обследования (ТО) — опрос жителей о их ежедневных перемещениях.
Каждая строка анкеты — одно перемещение (поездка) одного респондента.
У респондента может быть несколько перемещений в день.
Ключевые понятия:
- **OD-пара** (Origin-Destination) — пара "откуда-куда" (например "Дом - Работа")
- **Соцгруппа** — категория респондента (Работающий, Студент, Школьник, Другие)
- **Транспортная группа** — тип транспорта (ИТ, ОТ, Такси, Пешком и т.д.)
- **BW-модель** (Burt-Watts) — математическая модель распределения длительности поездок

## Входные данные

Все входные файлы лежат в `data/`.

### анкеты.xlsx — основная база
Столбцы: Статус, ID, Имя, Возраст, Пол, Социальный статус, Финансовое положение,
Затраты на транспор, Дата перемещений, Город, Адрес проживания, Координаты адреса,
Время отправления, Адрес отправления, Координаты отправления, Время прибытия,
Адрес прибытия, Координаты прибытия, Пункт отправления, Пункт прибытия,
Тип передвижения, Транспорт, Количество людей в машине,
Пешеходный подход к начальной остановке или парковке, Время ожидания транспорта,
Количество пересадок, Время ожидания между пересадками,
Пешеходный подход от конечной остановки или парковки,
Стоимость поездки или парковки, Комментарий

- **КРИТИЧНО: Координаты бывают перепутаны (lat/lon местами)** — обязательна автокоррекция!
  Правило: если первое число > 90 → это lon (Иркутск ≈ 104°E), порядок `lon, lat` — верный.
  Если первое число < 90 → это lat (Иркутск ≈ 52°N), порядок `lat, lon` — нужен swap.
  ```python
  def parse_lonlat(s):
      parts = str(s).replace(',', ' ').split()
      a, b = float(parts[0]), float(parts[1])
      return (a, b) if a > 90 else (b, a)  # всегда возвращает (lon, lat)
  ```
- Даты в формате YYYY-MM-DD
- Транспорт может содержать несколько значений через запятую — берём первый (`str.split(',').str[0].str.strip()`)

### Фильтрация по агломерации
Правильный способ — по полю `Город` (место проживания), а НЕ spatial join по координатам.
Причина: у человека из Иркутска могут быть поездки с началом в другом городе.
```python
agglo_map = {'Иркутск':'Иркутск', 'Ангарск':'Ангарск', 'Шелехов':'Шелехов',
             'Шелеховское городское поселение':'Шелехов'}
da = df[df['Город'].map(agglo_map).notna()]
```

### Справочники (xlsx)
- **Пары.xlsx** — столбцы = группы пунктов (Дом, Работа, Учеба, Школа, Прочее), значения = варианты названий
- **Соцгруппы.xlsx** — столбцы = соцгруппы (Работающий, Студент, Школьник, Другие), значения = варианты статусов
- **Транспорт.xlsx** — столбцы = транспортные группы (ИТ, Такси, ОТ, Служебный, СИМ, Пешком), значения = варианты

## Пайплайн обработки (4 этапа)

### Этап 1: Фильтрация (filter)
- Загрузить geojson и анкеты
- Пользователь выбирает: территории отправления, территории прибытия, тип дней (будни/выходные/все)
- Исправить координаты (автоопределение перепутанных lat/lon через попадание в полигоны)
- Spatial join: привязать точки к зонам
- Отфильтровать поездки: (start in origin AND end in dest) OR (start in dest AND end in origin)
- Экспорт: xlsx с отфильтрованными данными + базовая статистика

### Этап 2: Статус (status)
- Вход: xlsx после этапа 1
- Назначить соцгруппу каждому респонденту по справочнику Соцгруппы.xlsx
- Статистика: кол-во передвижений и респондентов по группам, средняя подвижность
- Экспорт: xlsx со статистикой + графики + листы по каждой соцгруппе

### Этап 3: OD-матрицы (od)
- Вход: xlsx после этапа 2
- Назначить OD-группы по справочнику Пары.xlsx
- Назначить транспортные группы по справочнику Транспорт.xlsx
- Построить OD-матрицы (абсолютные и на 1 респондента)
- Распределение по транспорту (общее и по каждой OD-паре)
- Экспорт: xlsx с матрицами, таблицами, графиками

### Этап 4: EVA-анализ (eva)
- Вход: xlsx после этапа 3
- Для каждой OD-пары и транспортной группы:
  - Распределение по часам отправления (0-23)
  - Распределение по длительности (бины по 5 мин, до 120 мин)
  - Подгонка BW-модели: φ(W) = E / (1 + exp(F - G·W)), BW(W) = 1 - 1/(1+W)^φ(W)
  - Метрики: MAE, RMSE, R²
- Статистика по доп. полям (пересадки, ожидание, стоимость и т.д.)
- Экспорт: xlsx с таблицами, графиками, формулами BW

## Что нужно ДОБАВИТЬ (новый функционал)

### GIS-экспорт
После каждого этапа (или по запросу) генерировать GIS-файлы:
- **GeoJSON** — точки отправления и прибытия с атрибутами
- **GeoPackage (.gpkg)** — все слои в одном файле (предпочтительный формат)
- **Shapefile (.shp)** — для совместимости
- **OD-линии** — линии от точки отправления к точке прибытия (desire lines)
- **Тепловые данные** — точки с весами для построения тепловых карт в QGIS

### QGIS-проект
Генерировать .qgz проект с:
- Слоями: границы территорий, точки отправления, точки прибытия, OD-линии
- Настроенной стилизацией (цвета по соцгруппам / транспорту)
- Подключёнными данными из output/

## Технические требования

- Python 3.10+
- Зависимости: см. requirements.txt
- CLI через стандартный input() (консольные меню)
- Все пути к данным настраиваются через config.yaml
- Кириллические имена файлов и столбцов — это нормально, не менять
- Код на русском не нужен — комментарии и docstring на английском, вывод для пользователя на русском

## Известные проблемы старого кода (legacy/)

1. Дублирование: normalize_excel_text, choose_input_xlsx, parse_columns_as_groups — копипаста в 3 файлах
2. Хрупкий парсинг координат — иногда не определяет перепутанные lat/lon
3. Нет обработки ошибок при чтении xlsx (bare except)
4. Скрипты работают только из корня папки с данными
5. Нет GIS-экспорта — только xlsx
6. xlsxwriter и openpyxl используются в разных скриптах — нужно унифицировать

## Как работать

1. Начни с чтения legacy/ файлов — там рабочая логика, её нужно переиспользовать
2. Создавай модули в src/ по одному, тестируй каждый
3. Приоритет: сначала заставь работать базовый пайплайн (этапы 1-4), потом добавляй GIS
4. Данные для тестов лежат в data/

python -m src.cli
